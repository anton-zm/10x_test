# Our online courses

Карточный каталог онлайн-курсов с живым поиском и фильтрацией по категориям.
Тестовое задание: pixel-perfect вёрстка по макету Figma на **HTML + SCSS + vanilla JS**, без библиотек и фреймворков.

🔗 **Демо:** https://10x-test-anton-zm.netlify.app
📁 **Репозиторий:** https://github.com/anton-zm/10x_test
🎨 **Макет:** [Figma — Test Task](https://www.figma.com/design/iuXLXifXgv4ZCzad0KeYKr/Test-Task)

---

## Возможности

- **Живой поиск** по названию курса — результаты фильтруются по мере ввода, без перезагрузки.
- **Фильтрация по категориям** — табы со счётчиками (`All 17`, `Marketing 4`, `Management 3`, `HR & Recruiting 5`, `Design 2`, `Development 3`). Комбинируется с поиском.
- **Load more** — карточки раскрываются порциями по 9.
- **Резиновый адаптив** от 320 до 1920px **без жёстких брейкпоинтов вёрстки**: размеры и отступы интерполируются через `clamp()`, число колонок сетки подбирается автоматически (`grid auto-fill`): 3 → 2 → 1.
- **Состояние «ничего не найдено»**.

## Стек и подход

| | |
|---|---|
| **Разметка** | Семантичный HTML, доступность (`aria-pressed`, `aria-label`, `alt`) |
| **Стили** | SCSS: модульная структура (`@use`), дизайн-токены, функция `fluid()` для резиновых значений, цвета бейджей из SCSS-карты |
| **Логика** | Vanilla JS (ES-модули): единое состояние UI, делегирование событий, рендер через шаблоны |
| **Именование** | [БЭМ](https://ru.bem.info/methodology/) |

### БЭМ

- Блоки: `header`, `filters`, `grid`, `tabs`, `search`, `card`, `decor-dots`, `decor-circles`, `decor-chevrons`.
- Элементы — через `__` (`grid__list`, `card__title`).
- Модификаторы — через одинарное подчёркивание (`tabs__button_active`, `card__badge_marketing`).

Разметка внутри `<main>`: `header` (заголовок) · `section.filters` (табы + поиск) · `section.grid` (карточки + «Load more»).

## Соответствие макету

Все значения сняты напрямую из Figma через REST API (а не «на глаз»):

- шрифт **Lato** (400 / 700 / 900);
- цвета: текст `#1e212c`, цена `#ff4242`, активный таб `#ff3f3a`, серый `#787a80` / `#9a9ca5`, рамки `#e5e8ed` / `#d7dadd`, жёлтый декор `#ffcf2d`;
- бейджи категорий: Marketing `#03cea4`, Management `#5a87fc`, HR `#f89828`, Design `#f52f6e`, Development `#7772f1`;
- сетка: контентная колонка 1230px, карточка 390×438, гэп 30px, внутренний паддинг карточки 24px;
- декоративные жёлтые акценты спозиционированы по координатам макета.

Фотографии курсов и декор экспортированы из макета (`assets/courses`, `assets/decor`).

## Структура

```
index.html                 разметка
src/
  scss/
    style.scss             точка входа
    _variables.scss         дизайн-токены + функция fluid()
    _base.scss              сброс, layout, глобальные стили
    blocks/                 БЭМ-блоки: header, filters, grid, tabs,
                            search, card, decor-*
  js/
    app.js                  состояние, поиск, фильтр, load more, рендер
    data.js                 курсы и категории
css/style.css              скомпилированный SCSS (для статик-деплоя)
assets/                    фото курсов и декор из макета
```

## Запуск локально

```bash
npm install     # установит sass и serve
npm start       # соберёт SCSS и поднимет сервер на http://localhost:3000
```

`npm start` — единственная команда запуска (сборка стилей + локальный сервер).

Дополнительно:
- `npm run build` — только сборка `css/style.css`;
- `npm run watch` — пересборка SCSS при изменениях.

> ES-модули не работают по `file://` — открывайте проект через локальный сервер.

## Деплой

Статический сайт. На Netlify задеплоен вручную из собранной папки.
Для сборки на стороне платформы:

- **Build command:** `npm run build`
- **Publish directory:** `.`

Скомпилированный `css/style.css` лежит в репозитории, поэтому сайт работает и без шага сборки.

## Принятые решения

- **17 курсов вместо 9.** В макете счётчики табов показывают 17 курсов, а на первом экране видно 9 + «Load more». Поэтому в данных 17 курсов: первые 9 — точно из макета (те же фото, цены, авторы), остальные 8 добавлены в том же стиле, чтобы счётчики категорий сходились и кнопка «Load more» работала осмысленно. Дополнительные курсы переиспользуют фото из макета.
- **Исправлены опечатки макета** в названиях: «Prduct» → «Product», «Recruting» → «Recruiting», двойной пробел в «HR  Management».
- **Декор** (жёлтые точки, круги, шевроны) показывается только на экранах ≥1600px — на меньших он мешал бы контенту. Это касается только декоративных элементов; сама вёрстка остаётся резиновой.
- **Заголовок** на экранах ≥1600px смещён влево на 26px — как в макете.
